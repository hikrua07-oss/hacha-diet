# サイト設計図 (Site Design Document)

## 1. プロジェクト概要
- **目的・コンセプト**: 2026年4月から2027年4月までの1年間で「マイナス10kg」を達成するためのパーソナルダイエット管理ツール。
- **ターゲットユーザー**: ユーザー自身（個人利用特化）。

## 2. コア機能要件
- **体重管理（短期・長期目標）**: 最終目標（1年で-10kg）だけでなく、モチベーション維持のための「今月の目標」といった短期的なマイルストーンも設定・可視化する機能。
- **食事管理（PFCバランス）**: 摂取カロリーだけでなく、タンパク質（P）、脂質（F）、糖質（C）の栄養素もしっかり記録・可視化する機能。
- **食事提案（AI活用）**: 過去の食事と現在のPFCバランスを分析し、不足している栄養素を補い、過剰なものを避けるような具体的な食事メニューの提案機能。
- **運動管理と提案**: 日々の歩数や手動入力の運動記録に加え、摂取カロリーやAIの分析に基づいた「おすすめの運動（例: 食後の軽いスクワット等）」を提案する機能。
- **リマインド機能**: 食事や運動などの記録忘れを防いだり、ダイエットのモチベーションや行動を促すための通知機能。

## 3. ページ構成 (サイトマップ)
- トップページ (ダッシュボード): 体重グラフ、今日の食事・運動サマリーの表示
- 食事管理ページ: おすすめメニューの提案、食事記録の入力・履歴
- 運動管理ページ: 運動記録の入力・履歴

## 4. デザイン要件・UI/UX
- **テーマカラー**: 落ち着いたブルーとグレーを基調とした色合い。危機的な状況（カロリーの大幅超過など）は、上品な赤色で警告表示。
- **フォント・タイポグラフィ**: Zen Kaku Gothic Newなど、すっきりと読みやすいモダンなフォント。
- **全体のテイスト**: 余白を活かした、シンプルで洗練されたクリーンなデザイン。

## 5. 技術スタック (予定)
- フロントエンド: HTML5, CSS3, Vanilla JavaScript (レスポンシブウェブデザイン対応)
- データ保存先: ローカルストレージ（ブラウザ内保存）、または必要に応じて外部API/DB
- 外部API: AI連携API (食事分析・提案用)

## 6. セキュリティと安全な公開・運用
- **APIキーの厳重管理**: AI連携（Geminiなど）でAPIキーを使用する場合、**HTMLやJavaScriptのコード内（フロントエンド）には絶対に直接書き込まない**こと。GitHub等にコードを公開した瞬間に悪用されるリスクがあるためです。
- **安全な実装方法**: APIキーが必要な処理は、バックエンドのサーバー（またはVercel/Netlifyなどのサーバーレス機能、Firebase Functions等）を経由して行い、キー自体は環境変数として安全な場所に隠す設計にします。
- **データ保護**: 各自のパソコンで動かすだけ（ローカルストレージ）であれば安全ですが、もし将来ネット上で使えるようにする場合は、他の人に見られないようなログイン機能やアクセス制限の仕組みが必要です。
